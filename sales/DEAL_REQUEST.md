# Zebepay Deal Request Guide

Use this guide before opening a public deal request.

## What To Share

Include:

- Buyer category.
- General use case.
- Package interest.
- Request for private contact.

For serious commercial discussions, use the approved private email/form after
the initial request.

Do not share:

- Passwords.
- Private keys.
- API tokens.
- Bank/provider details.
- BVN/NIN values.
- Customer data.
- Budget range.
- Confidential company strategy.
- Confidential contracts.
- Private documents.
- Internal infrastructure details.

## Package Tiers

### Tier 1 - Code Review Access

- Private read-only review access.
- Limited review period.
- No deployment support.

### Tier 2 - Source Code Buyer Package

- Full private repository access after approval.
- Buyer ZIP and SHA256 checksum.
- Setup, deployment, API, and buyer handoff docs.
- Basic setup support within approved scope.

### Tier 3 - White-Label And Setup Support

- Source package access.
- Branding customization.
- Deployment support.
- Provider integration guidance.
- Extended support window approved by MD.

## Deal Gate

Private source access is not granted until buyer approval, payment confirmation,
license acceptance, refund-policy acceptance, support-scope acceptance,
confirmed GitHub account, and delivery record completion.
